# Map Screen Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Adopt the shared design-system tokens and components on the Map screen (Google Maps + bottom sheets), the app's start destination, replacing every hand-rolled FAB, control pill, skeleton, drag handle, stat chip, gradient, invisible scrim, and raw brand-hex with the canonical shared primitives from Plans 01–03.

**Architecture:** Map UI lives in `commonMain` (`ui/home/map/`) even though Google Maps composables resolve via the androidMain `eu.buney.maps.*` actuals; this plan only edits the `commonMain` Kotlin and must not pull Android-only map APIs (`GoogleMap`, `CameraPosition`, `MapProperties`) into shared signatures. All shared components consumed here (`BrandFab`, `BrandGradients`, `FloatingControlSurface`, `FloatingIconButton`, `IconStatChip`, `SkeletonBox`/`SkeletonLine`, `DragDismissContainer`, `DragHandle`, `DismissScrim`, `StatePromptCard`, `GeocoderResultChip`) are built in Plans 01–03 and live in `ui/home/views/` or `ui/theme/`; this plan only calls them. Tokens (`Spacing`, `Shape`, `Elevation`, `BrandGradients`) come from `ui/theme/`.

**Tech Stack:** Kotlin Multiplatform, Compose Multiplatform, Material3, Google Maps (androidMain), Koin, Compose Resources.

**Sprint:** S2 · Primary Screens · **Plan 12 of 12**

**Depends on:** Plans 01–03 (tokens + components)

fun PlaceTileView(

@Composable
fun MapScaffold(

    modifier: Modifier = Modifier,
    onDismissCard: () -> Unit = {},
    fab: @Composable (() -> Unit)?,
    bottomCard: @Composable () -> Unit = {},

            visible = mapUiState is MapUiState.BottomSheetInfoMode,
            modifier = Modifier.align(Alignment.BottomCenter),
            enter = slideInVertically(

    }
